Overview

The Wizzyshop API allows you to call methods over the HTTPS protocol and sends the responds in REST JSON style.

To perform an action using the API, you need to select a calling convention, send a request to its endpoint specifying API Token (for some methods), a method and some arguments. The response will be received formatted as a JSON file.


Authentication

Most requests will no need authentication, but for some methods you will need a token that is send in the header Authorization.

Example 

POST /index.php/api/brand/follow HTTP/1.1
Host: wizzyshop.eddmi.com
Accept: application/json, text/plain, */*
User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 8_0 like Mac OS X) AppleWebKit/600.1.3 (KHTML, like Gecko) Version/8.0 Mobile/12A4345d Safari/600.1.4
Accept-Encoding: gzip, deflate, sdch
Accept-Language: en-US,en;q=0.8,pt-PT;q=0.6,pt;q=0.4,es;q=0.2,gl;q=0.2
Authorization: c25f735b86df0e8ex4f5e689aad2e71d
Content-Type: application/json
Cache-Control: no-cache
The token is generated each time the user logs in to the app.


Encoding

Use UTF-8 encoding when sending arguments to the API methods.
